# CloneBot 客户使用教程

## 1. 系统是做什么的

CloneBot 用于把 Telegram 源频道内容同步到目标频道。

- 用户号负责读取源频道内容。
- 官方 Bot API 负责发送到目标频道。
- 克隆任务用于批量同步历史内容。
- 监听任务用于实时同步新内容。

克隆任务完成后，如果源频道最新内容和目标频道最新内容一致，系统会自动进入监听。

## 2. 第一次使用前需要准备什么

1. 一个可读取源频道的 Telegram 用户号。
2. 一个或多个官方 Telegram Bot。
3. 把 Bot 加入目标频道。
4. 给 Bot 开启发送消息、发送图片、发送视频、发送文件等权限。
5. 准备源频道和目标频道名称，例如 `@channel_name`。

## 3. 推荐配置顺序

1. 账号管理
2. Bot 管理
3. 目标 Bot 绑定
4. 系统设置
5. 克隆任务
6. 监听任务
7. 运行日志

## 4. 账号管理

账号用于读取源频道。

新增账号时需要填写：

- 账号名称
- session 路径
- 代理信息，可为空
- 备注

账号必须能正常读取源频道，否则克隆任务和监听任务都无法获取源内容。

## 5. Bot 管理和目标绑定

Bot 管理用于维护官方 Bot Token。

建议添加 Bot 后先测试，确认 Token 可用。

目标绑定用于指定某个目标频道使用哪个 Bot 发送。如果目标频道没有绑定 Bot，系统会尝试使用可用的默认 Bot。

常见错误：

- `403 Forbidden`
- `bot is not a member of the channel chat`
- `not enough rights`

这些通常表示 Bot 没有加入目标频道，或者 Bot 没有足够权限。

## 6. 系统设置

系统设置控制全局发送节奏。

- 全局发送间隔：所有克隆和监听发送共用，避免发送太快。
- 重试次数：发送异常时最多重试几次。
- 重试等待：每次重试前等待的秒数。

## 7. 克隆任务

克隆任务用于同步历史内容。

主要字段：

- 任务名称：方便识别任务。
- 源频道：要读取内容的频道。
- 目标频道：要发送到的频道，格式为 JSON 数组，例如 `["@target1","@target2"]`。
- 账号 ID：用于读取源频道的用户号。
- 源频道开始内容链接：填写后从这条消息开始克隆，包含这条消息。
- 源频道结束内容链接：填写后克隆到这条消息结束，包含这条消息。
- 内容间隔分钟：每条内容发送完成后等待几分钟。
- 目标间隔秒：同一条内容发送到多个目标时，目标之间的间隔。
- 过滤关键词：命中后整条内容跳过。
- 替换词：把指定文字替换为新文字。
- 删除旧联系方式：自动删除包含手机号、Telegram 用户名、链接等联系方式的整行。
- Footer：发送时追加到内容末尾。
- 启用监听：克隆完成并确认最新内容一致后，自动进入监听。

说明：

- 纯文本、单媒体、相册都按“一条内容”处理。
- 单个目标失败不会影响其他目标继续发送。
- 如果媒体超过大小限制，系统会跳过对应内容。

## 8. 监听任务

监听任务用于实时同步源频道新消息。

监听任务使用用户号读取源频道，使用官方 Bot API 发送到目标频道。

主要操作：

- 启动：启用实时监听。
- 停止：停止实时监听。
- 编辑：修改源频道、目标频道、规则等配置。
- 删除：删除监听任务。
- 补齐：检查源频道和目标频道最新内容是否一致。

去重逻辑：

- 按目标频道单独去重。
- 同一个目标不会重复发送同一条源消息。
- 一个目标失败不会影响其他目标。

## 9. 过滤、替换和 Footer

过滤关键词：

- 内容命中关键词后，整条内容不会发送。

替换词：

- 把指定文字替换为新文字。

删除整行：

- 命中指定关键词后删除这一整行。

删除旧联系方式：

- 删除包含手机号、Telegram 用户名、链接等联系方式的整行。

Footer：

- 发送时追加到内容末尾。

## 10. 运行日志和发送缓存

运行日志用于查看系统运行状态和错误原因。

克隆任务和监听任务页面下方会显示最近发送缓存，最新记录在最上方。

发送缓存会展示：

- 时间
- 任务 ID
- 目标频道
- 源消息 ID
- 相册 ID
- 源链接
- 目标链接
- Bot 信息
- 成功或失败原因

## 11. 常见问题

目标没有收到内容：

- 检查 Bot 是否加入目标频道。
- 检查 Bot 是否有发送消息和媒体权限。
- 检查目标频道是否绑定了正确 Bot。

内容被删得太多：

- 检查是否开启“删除旧联系方式”。
- 检查删除整行规则是否过宽。
- 检查过滤关键词是否误命中。

监听没有生效：

- 检查监听任务是否启动。
- 检查账号是否能读取源频道。
- 检查 Bot 是否能发送到目标频道。

发送太快或太慢：

- 调整系统设置里的全局发送间隔。
- 调整克隆任务里的内容间隔分钟。

出现 403 权限错误：

- 把 Bot 加入目标频道。
- 给 Bot 开启发送消息、发送媒体权限。
- 检查目标频道和 Bot 绑定是否正确。
